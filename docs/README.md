# Documentation

## Structure

### architecture/
Canonical architecture specification (ARCHITECTURE_V6.md).
Defines all subsystem contracts, invariants, trust boundaries, and execution tiers.
This is the authoritative source for all implementation decisions.

### design/
Detailed design documents for specific subsystems and features.

### protocols/
Interface contract definitions, IPC protocols, and wire formats
for cross-subsystem communication.

### compatibility/
Compatibility target documentation, known-issue tracking,
and degradation behavior specifications.

### diagrams/
Architecture diagrams, data flow diagrams, and execution flow visualizations.
